Enhancement Two Narrative

The artifacts DatabaseManager.java, DataDisplayActivity.java, and WeightEntry.java are core components of my WeightWatcher Android application. Together, they showcase my ability to optimize algorithms, data structures, and implement efficient, secure, and maintainable coding practices.

  • DatabaseManager.java: I improved performance and security by using projections to retrieve only necessary data, parameterized queries to prevent SQL injection, and a caching mechanism to reduce database calls.
  • DataDisplayActivity.java: I enhanced algorithmic efficiency, introducing methods to find insertion points for data, refining sorting strategies, and optimizing RecyclerView updates for smoother UI interactions.

Across all three artifacts, I applied comprehensive exception handling, ensured proper resource management, and added clear documentation. These steps reinforced data integrity, reduced crashes, and improved code readability, aligning with industry best practices.

Enhancement Three Narrative

In this third enhancement of my WeightWatcher Android application, I optimized the database schema, improved input validation, and strengthened security measures to enhance performance, usability, and resilience. The main artifacts—DatabaseHelper.java, SmsPermissionActivity.java, DatabaseManager.java, and DataDisplayActivity.java—demonstrate these improvements:

  • DatabaseHelper.java:Introduced constraints (NOT NULL, UNIQUE, CHECK), foreign keys with cascading actions, and indexes to enforce data integrity, maintain referential integrity, and improve query performance.
  • SmsPermissionActivity.java:Implemented phone number validation with regex, ensuring valid user input while providing immediate feedback. This improved both security and user experience.
  • DatabaseManager.java and DataDisplayActivity.java:Adjusted logic to handle NULL values correctly, preventing errors when prompting users for goal weights. Enhanced exception handling and resource management ensured stable and secure data operations.

These changes align with professional standards and demonstrate my proficiency in database optimization, secure coding, and user-centric design. They supported the course outcomes related to building professional-quality software (Outcome 2), optimizing algorithmic data handling (Outcome 3), leveraging innovative techniques (Outcome 4), and anticipating security challenges (Outcome 5).

Throughout this process, I learned that even small schema changes require careful testing and logic adjustments. Introducing NULL-based logic for goal weights revealed interdependencies that demanded thorough debugging. I also confirmed that balancing security (like input validation) with usability can significantly improve user satisfaction without compromising safety. Ultimately, these enhancements reinforced the value of incremental development, code reviews, and strategic debugging to achieve a reliable and efficient application.
